# A Large-Scale Study on the Development and Issues of Multi-Agent AI Systems - Analysis

## 1. Overview

이 논문은 오픈소스 멀티 에이전트 AI 시스템(MAS)의 개발 및 유지보수 관행에 대한 최초의 대규모 실증 연구이다. LangChain, CrewAI, AutoGen, Haystack, LlamaIndex, Semantic Kernel, Letta, SuperAGI 등 GitHub에서 가장 활발한 8개 프레임워크를 대상으로, 42,267개의 고유 커밋과 4,731개의 해결된 이슈를 분석한다. LLM 기반 멀티 에이전트 시스템이 빠르게 확산되고 있지만, 이들이 실제로 어떻게 진화하고 유지보수되는지에 대한 실증적 이해는 부족했다.

저자들은 소프트웨어 공학의 소프트웨어 저장소 마이닝(MSR) 방법론을 MAS 생태계에 적용하여, 커밋 활동 패턴, 코드 변경 유형, 이슈 분류 및 해결 시간을 체계적으로 분석한다. 커밋 분류에는 fine-tuned DistilBERT 모델을 사용하고, 이슈 토픽 분석에는 BERTopic을 활용한다.

핵심 발견으로, MAS 생태계에서 세 가지 개발 프로파일(sustained, steady, burst-driven)이 식별되었으며, perfective 커밋이 40.8%로 corrective(27.4%)와 adaptive(24.3%)를 크게 상회한다. 이슈 측면에서는 버그(22%), 인프라(14%), 에이전트 코디네이션(10%)이 주요 관심사이며, 2023년을 기점으로 모든 프레임워크에서 활동이 급증했다.

이 연구의 의의는 MAS 생태계의 성숙도와 취약성을 동시에 보여준다는 점이다. 빠른 기능 확장이 이루어지고 있지만 테스트 인프라, 문서 품질, 유지보수 관행의 개선이 장기적 신뢰성 확보를 위해 필요하다는 점을 데이터로 입증한다. 특히 MAS의 핵심 차별화 기능인 에이전트 관련 이슈가 전체의 10%에 불과하고, 기초적인 소프트웨어 품질 문제(버그, 인프라)가 더 빈번하다는 점은 생태계의 현 단계를 잘 보여준다.

## 2. Core Section

### TL;DR

> 8개 주요 오픈소스 멀티 에이전트 AI 프레임워크의 42K+ 커밋과 4.7K+ 이슈를 분석한 최초의 대규모 실증 연구로, 생태계가 기능 중심 개발(perfective 40.8%)에 치중하며 2023년을 기점으로 급성장했으나, 테스트/인프라 취약성과 프레임워크 간 성숙도 불균형이 존재함을 밝혔다.

-> 상세: [tldr.md](./2601.07136-details/tldr.md)

### Core Contributions

1. **최초의 MAS 대규모 실증 연구**: 8개 프레임워크, 42K+ 커밋, 4.7K+ 이슈를 포괄하는 최초의 체계적 분석 -> 기존에 알고리즘/아키텍처 중심이던 MAS 연구에 소프트웨어 공학 관점을 최초로 도입
2. **세 가지 개발 프로파일 식별**: Sustained(Haystack), Steady(Semantic Kernel), Burst-driven(SuperAGI) -> 생태계 성숙도와 프로젝트 건강성 평가의 프레임워크 제공
3. **커밋/이슈 유형의 정량적 분석**: Perfective 40.8% 우세, 버그 22% 최다 이슈 -> 기능 확장 우선, 유지보수 부족이라는 생태계 특성 규명
4. **2023년 전환점 발견**: 커밋/이슈 모두 2023년부터 급증하는 inflection point 확인 -> LLM 확산과 MAS 생태계 성장의 상관관계 실증

## 5. Critique & Related Works

### Expert Critique

#### Strengths
1. **선구적 연구 영역**: MAS에 MSR 방법론을 적용한 최초 연구로, 알고리즘/아키텍처 중심이던 기존 연구에 소프트웨어 공학적 관점을 도입
2. **대규모 데이터**: 42K+ 커밋, 4.7K+ 이슈라는 충분한 규모의 데이터로 통계적 유의미성 확보
3. **다층적 분석**: 커밋 활동, 코드 변동, 커밋 유형, 이슈 분류, 해결 시간, 토픽 모델링까지 다양한 관점에서 분석
4. **실용적 시사점**: 생태계의 취약성(테스트 인프라 부족, 불균등한 성숙도)을 구체적 데이터로 지적

#### Limitations
1. **시간적 스냅샷**: 데이터 수집 시점의 스냅샷이므로, 빠르게 변화하는 생태계에서 유효 기간이 짧을 수 있음. 특히 2025-2026 MAS 생태계 변화(OpenAI Agents SDK, Claude MCP 등)를 반영하지 못함
2. **오픈소스 편향**: GitHub 오픈소스만 분석하여, Claude/GPT 기반 독점 MAS 생태계를 배제. 실제 프로덕션 환경의 관행과 차이가 있을 수 있음
3. **인과관계 부재**: 개발 프로파일이나 이슈 패턴의 "왜"를 설명하지 못함. 예: SuperAGI의 burst-driven 패턴이 프로젝트 실패인지 전략적 피벗인지 불명확
4. **커밋 분류의 한계**: DistilBERT 기반 자동 분류의 정확도 검증이 부분적이며, 커밋 메시지의 품질에 의존
5. **정성적 분석 부족**: 코드 품질, 설계 결정의 합리성, 개발자 경험 등 정성적 측면을 다루지 않음

#### 2026 Perspective
- **Still Valid**: 생태계 불균형, 기능 중심 개발 경향, 인프라 문제의 지속성
- **Outdated**: 2025-2026년 급변한 MAS 생태계 (OpenAI Agents SDK, Anthropic MCP, Google A2A 프로토콜 등)가 미반영
- **Missing**: MCP(Model Context Protocol) 기반 도구 통합 패러다임, 에이전트 안전성/정렬 관련 이슈, multi-modal agent 관련 개발 트렌드

### Related Works

1. **Guo et al. (2024) "LLM-based Multi-Agents: A Survey"** - MAS의 아키텍처/알고리즘 분류 체계를 제공하여, 본 논문의 소프트웨어 공학적 분석과 보완적 관점 제공 - [IJCAI-24](https://arxiv.org/abs/2402.01680)
2. **Xi et al. (2025) "The Rise and Potential of LLM-based Agents: A Survey"** - LLM 에이전트의 포괄적 서베이로, 개별 에이전트에서 MAS까지의 스펙트럼 이해에 도움 - [Science China](https://arxiv.org/abs/2309.07864)
3. **AgentBench (Liu et al., 2023)** - MAS 평가 벤치마크로, 본 논문에서 다룬 이슈 유형(에이전트 코디네이션, 함수 호출 등)의 실제 성능 관점 제공 - [arXiv:2308.03688](https://arxiv.org/abs/2308.03688)
4. **Park et al. (2023) "Generative Agents"** - 멀티 에이전트 상호작용의 사회적 역학 탐구로, 본 논문이 식별한 "agent coordination" 이슈의 기술적 배경 이해에 도움 - [UIST '23](https://arxiv.org/abs/2304.03442)
